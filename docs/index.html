Double-click Setup · once

This downloads everything the app needs. A black window opens, prints some text for a minute, and tells you when it's finished. You only ever do this once.

First time only: macOS may say it “can't verify the developer.” That's expected for an in-house tool. Instead of double-clicking, right-click the Setup file → Open → Open. After that one time, double-click works normally.
Wait for “Setup complete.” Then close the window — you're set up for good.

Double-click Start Downloader · anytime

This is the only thing you do day-to-day. A window opens and your browser pops up with the downloader ready to go. Paste your links and download.

Keep the window open while you work. When you're done, close it to shut the app down.

3✓

Install the app · one time

Copy the whole block, paste it all at once, press return. It may take a minute and print a lot of text — that's fine.

Terminal — paste all of this
cd ~/Downloads/YouTube_Downloader_Clean_Source
python3 -m venv venv
source venv/bin/activate
python3 -m pip install flask yt-dlp -i https://pypi.org/simple/
mkdir -p bin
curl -sL -o ffmpeg.zip https://evermeet.cx/ffmpeg/getrelease/zip
unzip -q ffmpeg.zip -d bin/
rm ffmpeg.zip
chmod +x bin/ffmpeg
When the cursor returns with no red ERROR lines, you're done forever.

If something looks wrong

It says it “can't verify the developer” +
Expected for an in-house tool. Right-click the file → Open → Open (instead of double-clicking). You only do this once per file. If your Mac is locked down by IT and offers no “Open” option, that's a Santa/MDM block — ask IT to allowlist the app, or use the Manual method, which runs through tools already approved on your Mac.
It opened as text instead of running +
The file lost its “runnable” flag when it was downloaded. Open Terminal, type chmod +x (with the trailing space), then drag the file onto the Terminal window and press return. Double-click works after that. Running Setup this way also re-arms Start Downloader automatically.
It says “pip not found” +
Use python3 -m pip instead of pip. The Manual block above already does this.
It says “Address already in use” +
The app is already running from earlier — just open 127.0.0.1:5001 in your browser. No need to start it again.
A download fails with a “403” error +
Already handled in this version. If it keeps happening, tell whoever set the app up for you.
