1✓

Open it · first time

Double-click YouTube Downloader. macOS blocks unsigned tools the first time, so it won't open yet — here's the one-time unlock.

  1. On the “Apple could not verify…” message, click Done.
  2. Open System Settings ( menu → System Settings, or press ⌘ Space and type “Privacy”).
  3. Go to Privacy & Security and scroll down to the Security section.
  4. Click Open Anyway (circled below), then enter your Mac password.

This is the screen you're heading to — click the glowing button.

No password works, or no Open Anyway button (a locked-down Mac)? Open Terminal, type xattr -dr com.apple.quarantine and a space, drag the folder onto the window, press return — then open YouTube Downloader.
It then sets up (about a minute the first time) and your browser opens. Keep the window open while you work.

3✓

Install · one time

Type cd and a space, then drag the folder you unzipped onto the Terminal window and press return.

Then paste this block and press return:

Terminal
python3 -m venv venv
source venv/bin/activate
python3 -m pip install flask yt-dlp -i https://pypi.org/simple/
mkdir -p bin
curl -sL -o ffmpeg.zip https://evermeet.cx/ffmpeg/getrelease/zip
unzip -q ffmpeg.zip -d bin/
rm ffmpeg.zip
chmod +x bin/ffmpeg
No red ERROR lines means it worked.

If something looks wrong

“Apple could not verify… free of malware” +
Normal for an unsigned in-house tool. To allow it (once):
  1. Click Done.
  2. System Settings → Privacy & Security → scroll to Security.
  3. Click Open Anyway, enter your password.
No admin rights, or no button? Terminal: xattr -dr com.apple.quarantine + a space, drag the folder in, return. If even that's blocked, your Mac is IT-locked — they'll need to allowlist it, or use the Manual tab.
The “Open Anyway” button isn't there +
It only appears for about an hour after the block, and only after you've tried to open the file. Double-click YouTube Downloader again, then go back to Privacy & Security and it'll be waiting.
The folder has lots of files in it +
Normal — the download includes the app's source too. You only need YouTube Downloader; ignore the rest.
It opened as text instead of running +
Terminal → chmod +x (trailing space) → drag the file in → return. Double-click works after.
Download fails with “403” +
Handled in this version. If it persists, flag it to whoever set this up.
